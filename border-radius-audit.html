🚨 Platform-Wide Critical Issues

These inconsistencies span the entire NYT Games platform, affecting user experience across Crosswords, Spelling Bee, Wordle, Letter Boxed, Tiles, Sudoku, Statistics, Leaderboards, and Archive.

🔘 Border Radius

  • 12 different values across platform buttons
  • Modal inconsistency confuses users switching games
  • No design system guidance for developers
  • Brand fragmentation across game portfolio

📝 Typography

  • 6+ font family declaration patterns
  • 50+ font sizes with no systematic scale
  • Cross-game confusion from varied text hierarchy
  • Accessibility issues from inconsistent sizing

🎨 Colors

  • Color conflicts between games (same name, different hex)
  • 30+ gray variations causing visual inconsistency
  • Theming problems across platform features
  • Brand dilution from inconsistent game colors

📏 Spacing

  • Hundreds of hardcoded px values platform-wide
  • Linting bypassed in many game directories
  • Responsive issues from inconsistent units
  • Maintenance burden across 9 products

💡 Comprehensive Standardization Roadmap

🔘 Border Radius System

--radius-none: 0;
--radius-sm: 4px; - Cards, small components
--radius-md: 8px; - Modals, containers
--radius-lg: 16px; - Large buttons
--radius-xl: 24px; - Prominent buttons
--radius-full: 9999px; - Pills, fully rounded

📝 Typography Scale

--text-xs: 12px; - Captions, labels
--text-sm: 14px; - Small text
--text-base: 16px; - Body text
--text-lg: 18px; - Large text
--text-xl: 20px; - Headings
--text-2xl: 24px; - Large headings
--text-3xl: 32px; - Display text

🎨 Color Consolidation

Single Source: Consolidate colors.scss & colors.ts
Gray Scale: Reduce 30+ grays to 8-10 systematic values
Game Colors: Fix conflicting hex values
Semantic Names: Use purpose-based naming

📏 Spacing Enforcement

Enable Linting: Remove stylelint-disable comments
Token Migration: Replace hardcoded px values
Consistent Units: Standardize on rem/em
Design Tokens: Expand spacing scale

🚀 Platform-Wide Implementation Strategy

NYT Games Platform Scope: These inconsistencies affect the entire NYT Games ecosystem, impacting millions of users across 9 major game products and their associated features.

Phase 1 - Platform Foundation
• Audit all 9 game products
• Consolidate shared design tokens
• Fix critical color/spacing conflicts
• Establish design system governance
Phase 2 - Shared Components
• Unify button systems across games
• Standardize modal implementations
• Create platform typography scale
• Consolidate navigation patterns
Phase 3 - Game-by-Game Migration
• Migrate Crosswords (highest traffic)
• Update Wordle & Spelling Bee
• Standardize remaining games
• Enable platform-wide linting
Phase 4 - Platform Maintenance
• Cross-game design system testing
• Platform documentation
• Team training & governance
• Continuous monitoring
💡 Strategic Recommendation

Given the platform scope, consider establishing a Games Design System Team to own standardization across all products, ensuring consistent user experience and reducing development overhead for the entire NYT Games portfolio.
