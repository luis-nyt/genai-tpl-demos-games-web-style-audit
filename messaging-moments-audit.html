Error Messages
Connection/Network Errors
Offline: "Oh no! You're offline. Find a connection and come back."
Generic: "Oops! Something went wrong. Please try again later."
Missing Data: "Oops! We're missing data for this puzzle. Please test after the data has been added."
Launch Date: "Oops! Date precedes the launch date of [date]. Please test a later puzzle."
Spelling Bee Input Errors
Too Short: "Too short" (words must be 4+ letters)
Bad Letters: "Bad letters" (contains invalid letters)
Missing Center: "Missing center letter"
Not in List: "Not in word list"
Already Found: "Already found"
Too Long: "Too long" (over 18 characters)

1 guess: "Genius"
2 guesses: "Magnificent"
3 guesses: "Impressive"
4 guesses: "Splendid"
5 guesses: "Great"
6 guesses: "Phew"
Failed game: Shows the solution word in caps

Pangram: "Pangram!" (uses all letters, +7 bonus points)
7+ points: "Awesome!"
2-6 points: "Nice!"
1 point: "Good!"
Plus floating point display (e.g., "+5")

Short words (under 7 letters): "Nice!" or "Awesome!" (based on previous success)

Help & Tutorial Content
Wordle Instructions
Main Rule: "Guess the Wordle in 6 tries."
Requirements: "Each guess must be a valid 5-letter word."
Feedback: "The color of the tiles will change to show how close your guess was to the word."
Examples: "W is in the word and in the correct spot" / "I is in the word but in the wrong spot" / "U is not in the word in any spot"
Spelling Bee Instructions
Main Rule: "Create words using letters from the hive."
Requirements: "Words must contain at least 4 letters" / "Words must include the center letter" / "Letters can be used more than once"
Scoring: "4-letter words are worth 1 point each" / "Longer words earn 1 point per letter" / "Each puzzle includes at least one 'pangram' which uses every letter. These are worth 7 extra points!"
Schedule: "New puzzles are released daily at 3 a.m. ET."
Connections Instructions
Main Rule: "Find groups of four items that share something in common."
Gameplay: "Select four items and tap 'Submit' to check if your guess is correct."
Limit: "Find the groups without making 4 mistakes!"
Examples: "FISH: Bass, Flounder, Salmon, Trout" / "FIRE ___: Ant, Drill, Island, Opal"

Key Findings & Recommendations

Strengths:

  • Comprehensive Coverage: All major messaging patterns are implemented
  • Consistent Voice: Messages use friendly, encouraging tone across games
  • Shared Systems: Good reuse of tooltip, modal, and toast systems
  • Accessibility: Consistent ARIA patterns and screen reader support
  • Animation Consistency: Similar animation patterns across games
  • Progressive Feedback: Different success messages based on performance level

Message Content Patterns:

  • Error Messages: Use "Oops!" frequently, provide clear next steps
  • Success Messages: Progressive celebration (Good ‚Üí Nice ‚Üí Awesome ‚Üí Genius)
  • Instructions: Clear, concise rules with concrete examples
  • Offline Messaging: Consistent "Find a connection and come back" theme
  • Badge Messaging: Emphasizes accomplishment and discovery

Areas for Improvement:

  • Message Standardization: Some games have slightly different error wording
  • Animation Standardization: Could benefit from shared animation utilities
  • Mobile Optimization: Some tooltip positioning could be improved for mobile
  • Internationalization: Hard-coded strings could be moved to i18n system

Design System Opportunities:

  • Centralized message content library with consistent voice
  • Standardized error message hierarchy and types
  • Unified success message progression system
  • Shared animation timing and easing patterns
  • Consolidated accessibility utilities and ARIA patterns
