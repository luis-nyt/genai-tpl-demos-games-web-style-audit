# TPL Web 2.0 Deprecation Notices

The next major version of `@nyt/tpl` (“TPL Web 2.0”) will migrate to NYT's new CSS standard, which uses **CSS Modules and Sass**.

At minimum, this involves converting existing component styles. Some TPL Web 1.0 features depend upon a CSS-in-JS “runtime”, which the new CSS standard intentionally lacks. **This means we must also carefully replace or remove certain features.**

Please refer to the [**TPL 2.0 Release Plan**](https://docs.google.com/document/d/12T5bEefMJDYoLY8gtg0cokrNco53h5t8Ofj8rn5uaV4/edit?tab=t.2tb2mnrxg2u6) on Google Docs for more information. We've reproduced the list of [deprecated exports](#deprecated-tpl-web-symbols--component-props) and [components with deprecated props](#tpl-web-components-with-deprecated-props) below for convenience.

For more info on the new CSS standard, please see [Styling - Times Tech Standards](https://standards.tech.nyt.net/standards/web/styling/) and [RFC: Adopting CSS Modules with Sass as NYT Standard](https://docs.google.com/document/d/1KghooikS8RzyKa84N4Fh_tQ0MJpYRGhf1bHGDjfCUik/edit).
